供应链库存周转率分析与优化
项目背景：
以电商供应链为业务背景，针对 3 个核心仓库、50 个 SKU、5000+ 条销售记录，独立完成从数据清洗到优化建议的全流程分析。分析时间段为 2026 年 Q2（4 月初至 6 月末）。

项目目标：
计算各仓库及各 ABC 类商品的库存周转率
识别滞销品，量化其资金占用
提出可落地的库存优化方案

分析思路：
数据获取：从 MySQL 读取商品表、库存快照表、销售表等
数据清洗：用 Pandas 处理空值和异常值
周转率计算：按仓库 × ABC 分类 × 月度三个维度交叉计算
滞销品识别：根据库龄阈值每日识别滞销品，统计其金额占比
可视化分析：Python 图表 + Power BI 动态看板
优化建议：针对不同类别商品提出差异化策略

核心结论：
通过“分仓独立 ABC 分类 + 月度周转率追踪 + 滞销品库龄预警”的分析框架，有效定位了库存管理中的瓶颈环节。将分析结果转化为差异化补货策略和滞销品清理方案，预期可显著释放积压资金、提升整体周转效率。
